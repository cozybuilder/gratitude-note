## 1. 프로젝트 비전

### 왜 감사노트를 만들었는가

매일 3가지 감사한 일을 기록하는 습관은 심리학적으로 행복감과 삶의 만족도를 높이는 효과가 입증되어 있습니다. 그러나 기존의 감사일기 앱들은 지나치게 복잡하거나, 불필요한 소셜 기능을 포함하거나, 구독료를 요구합니다.

감사노트는 **가장 단순하고 순수한 감사일기**를 지향합니다. 매일 3가지를 쓰고, 기록이 쌓이고, 습관이 형성되는 것. 그것만으로 충분합니다.

### 서비스 철학

- **기록은 로컬에**: 사용자의 감사 기록은 사용자의 것입니다. 서버로 전송되지 않습니다.
- **단순함이 지속성을 만든다**: 복잡한 기능은 오히려 습관 형성을 방해합니다.
- **AI는 보조, 사람이 주인공**: AI는 축하 메시지 정도에만 사용하며, 기록의 주체는 언제나 사용자입니다.
- **무료로 충분히 가치 있게**: 핵심 기능은 완전 무료입니다. 광고 수익으로 운영비를 충당합니다.

### 타 서비스와의 차별점

| 항목 | 감사노트 | 일반 일기 앱 |
|------|----------|-------------|
| 감사 3가지 특화 | ✅ 핵심 구조 | ❌ 자유 형식 |
| 데이터 로컬 저장 | ✅ 서버 없음 | ❌ 클라우드 동기화 |
| 습관 배지 시스템 | ✅ 7·30·60·90·180·365일 | △ 일부만 |
| 완전 무료 | ✅ 광고만 | ❌ 구독 요구 다수 |
| PWA + Android | ✅ 동시 지원 | △ |
| AI 축하 메시지 | ✅ 배지 획득 시 1회 | ❌ |

## 2. 핵심 원칙

### 원칙 1: 단순함 우선

- 화면 구성은 최소화합니다. 기능을 추가할 때마다 "정말 필요한가"를 먼저 묻습니다.
- 폼은 감사 3가지 + 기분 선택으로 고정합니다. 확장하지 않습니다.
- 입력 글자 수는 최대 40자로 제한합니다 (공유 카드 품질 보장 목적).

### 원칙 2: 습관 형성 우선

- 연속 기록(streak)이 서비스의 핵심 지표입니다.
- 배지 시스템은 외적 동기를 부여합니다. 배지는 한 번 획득하면 영구히 유지됩니다 (streak이 낮아져도 삭제되지 않음).
- 새벽 4시 기준의 날짜 경계(`gratitudeDate`)로 밤 늦게 쓰는 사용자를 배려합니다.
- 알림(리마인더) 기능으로 매일 기록을 독려합니다.

### 원칙 3: AI는 보조 기능

- AI는 배지 획득 시 1회 축하/회고 메시지 생성에만 사용합니다.
- 일상 감사 기록의 응원 메시지(`generateAiMessage`)는 로컬 템플릿입니다. 이 함수는 수정하지 않습니다.
- AI API는 기본 OFF(`ENABLE_BADGE_AI_API = false`)입니다.
- API 키는 절대 프론트엔드 번들에 포함시키지 않습니다.

### 원칙 4: 운영비 최소화

- 서버(백엔드)를 운영하지 않습니다.
- 모든 사용자 데이터는 기기의 `localStorage`에 저장됩니다.
- AI API는 배지 획득이라는 극소 빈도 이벤트에만 사용하므로 비용이 미미합니다.
- Vercel 무료 플랜으로 운영합니다 (API Function 포함).

### 원칙 5: 사용자 가치 우선

- 데이터 백업/복원 기능을 제공합니다 (JSON 파일 내보내기/가져오기).
- 공유 카드 기능으로 SNS 공유를 지원합니다.
- 다크 모드를 지원합니다.
- 개인정보처리방침을 앱 내에 포함합니다.

## 4. 기술 스택

### Frontend

| 항목 | 버전 | 비고 |
|------|------|------|
| React | 19.x | UI 프레임워크 |
| Vite | 8.x | 빌드 도구 |
| TypeScript | 6.x | 타입 안전성 |
| Tailwind CSS | v4 | 스타일링 |
| React Router | 7.x | 클라이언트 라우팅 |

### Storage

- **`localStorage`** — 모든 사용자 데이터 저장
- 서버 DB 없음, 클라우드 동기화 없음
- 백업/복원은 JSON 파일 내보내기/가져오기로 대체

### PWA

- **vite-plugin-pwa** — Service Worker 자동 생성
- `registerType: 'autoUpdate'`
- 홈 화면 추가 지원, 오프라인 캐싱

### Android

- **Capacitor 8.4.0** — WebView 기반 하이브리드 앱
- `appId: com.cozybuilder.gratitudediary`
- `appName: 감사일기`
- `webDir: dist`
- 빌드: `npm run build` → `npx cap sync android` → Android Studio

### Vercel

- **정적 호스팅**: `dist/` 빌드 결과물 서빙
- **Serverless Function**: `api/badge-celebration.ts` (Node.js Runtime)
- **라우팅**: `vercel.json` routes 설정
  ```json
  { "src": "/api/(.*)", "dest": "/api/$1" },
  { "src": "/(.*)",     "dest": "/index.html" }
  ```
- **도메인**: https://gratitude-note-theta.vercel.app
- **배포**: GitHub master push 시 자동 배포

### OpenAI (현재 기본 OFF)

- **모델**: gpt-4o-mini
- **용도**: 배지 획득 시 1회 축하/회고 메시지 생성
- **호출 방식**: 프론트 → Vercel Function → OpenAI API
- **API 키 위치**: Vercel 대시보드 환경변수 `OPENAI_API_KEY` (서버 전용)
- **프론트 환경변수**: `VITE_BADGE_AI_ENDPOINT` (Function URL, 키 아님)
- **플래그**: `ENABLE_BADGE_AI_API = false` (src/utils/ai.ts)

### 환경변수 구조

```bash
# .env.local (Git 미포함, 로컬/Android 빌드용)
VITE_ADMOB_APP_ID=            # Google AdSense Publisher ID
VITE_ADMOB_BANNER_ID=         # AdSense 배너 슬롯 ID
VITE_BADGE_AI_ENDPOINT=       # Vercel Function URL (AI 활성화 시 설정)

# Vercel 대시보드 (서버 환경변수)
OPENAI_API_KEY=sk-...         # OpenAI API 키 (절대 프론트에 노출 금지)
```

## 5. 아키텍처 결정사항

### 왜 로컬 저장소를 선택했는가

**결정**: 모든 사용자 데이터를 `localStorage`에만 저장.

**이유**:
1. **개인정보 보호**: 사용자의 감사 기록은 민감한 개인 데이터입니다. 서버로 전송하지 않음으로써 유출 위험을 원천 차단합니다.
2. **운영비 제로**: DB 서버 비용이 없습니다.
3. **오프라인 지원**: 인터넷 없이도 앱이 완전히 동작합니다.
4. **구현 단순성**: 인증, 동기화, 충돌 해결 등의 복잡성이 없습니다.

**트레이드오프**:
- 기기 교체 시 데이터 이전이 불편합니다 → JSON 백업/복원으로 해결
- 복수 기기 동기화 불가 → MVP 단계에서는 불필요한 기능으로 판단

### 왜 서버를 두지 않았는가

**결정**: 백엔드 서버 없음. Vercel Serverless Function만 사용.

**이유**:
1. 사용자 데이터를 서버에 저장하지 않으므로 서버가 불필요합니다.
2. AI API 키 보안을 위한 최소한의 서버 기능만 Serverless Function으로 해결합니다.
3. 서버 유지 비용·장애 대응 부담을 없앱니다.
4. Vercel Serverless Function은 사용한 만큼만 과금됩니다.

### 왜 AI를 기본 OFF로 했는가

**결정**: `ENABLE_BADGE_AI_API = false`가 기본값.

**이유**:
1. **비용 통제**: AI API는 사용량에 따라 비용이 발생합니다. 사용자가 늘어나도 기본 OFF면 예상치 못한 비용 급증이 없습니다.
2. **로컬 템플릿으로 충분**: 배지별 5개의 정성스러운 한국어 메시지로도 사용자 경험이 충분합니다.
3. **인프라 보존**: API 코드와 Serverless Function은 그대로 유지합니다. `ENABLE_BADGE_AI_API = true`로 전환하는 즉시 AI 메시지가 활성화됩니다.
4. **점진적 도입**: 향후 유료 플랜, 특별 이벤트, A/B 테스트 등 다양한 방식으로 활성화할 수 있습니다.

### 왜 Capacitor를 선택했는가

**결정**: React Native나 Flutter 대신 Capacitor 사용.

**이유**:
1. 이미 완성된 React 웹 앱을 그대로 Android에 배포할 수 있습니다.
2. 코드베이스 단일화 — 웹과 Android를 같은 코드로 유지합니다.
3. PWA와 Android 앱을 동시에 제공합니다.

## 7. 절대 변경하면 안 되는 것

### 7-1. localStorage 키 (데이터 호환성)

```
gratitude_notes          ← 절대 변경 금지
gratitude_achievements   ← 절대 변경 금지
theme_mode               ← 절대 변경 금지
onboarding_done          ← 절대 변경 금지
```

이 키들을 변경하면 기존 사용자의 모든 데이터가 초기화됩니다.

### 7-2. 핵심 함수 (기존 데이터 호환성)

```ts
generateAiMessage()          ← 수정 금지 (기존 저장 데이터와 연관)
calcStreak()                 ← streak 계산 로직 변경 시 사전 검토 필수
getGratitudeDate() / 4AM     ← 날짜 경계 변경 금지 (연속 기록 깨짐)
```

### 7-3. Android 앱 식별자

```
appId: com.cozybuilder.gratitudediary  ← 절대 변경 금지
```

Play Store 등록 후 변경 불가. 변경 시 새 앱으로 재등록해야 합니다.

### 7-4. 보안 원칙

```
VITE_OPENAI_API_KEY 사용 금지     ← PM 명시적 금지
프론트엔드 번들에 OpenAI 키 포함 금지
키스토어 파일 Git 커밋 금지
키스토어 비밀번호 GitHub 저장 금지
```

### 7-5. PM 의사결정 기준

1. **단순함 > 기능 수**: 기능 추가 전 "없어도 되는가"를 먼저 물어볼 것
2. **사용자 데이터 보호**: 서버 전송이 필요한 기능은 철저한 검토 후 PM 승인
3. **운영비 통제**: AI API 등 비용 발생 기능은 PM이 직접 ON/OFF 결정
4. **하위 호환성 보장**: 기존 사용자 데이터는 어떤 경우에도 보호

```js
// 오늘(YYYY-MM-DD) 기준 과거 6일치 주입 후 새 노트 작성 → 7일 달성
const days = ['2026-06-02','2026-06-03','2026-06-04','2026-06-05','2026-06-06','2026-06-07']
const notes = days.map((d, i) => ({
  id: 'test-' + i,

  gratitudeDate: d,
  gratitude1: '테스트1', gratitude2: '테스트2', gratitude3: '테스트3',
  mood: 'good', aiMessage: '테스트'
}))
localStorage.setItem('gratitude_notes', JSON.stringify(notes))
localStorage.setItem('gratitude_achievements', '[]')
location.reload()
// → 오늘 날짜는 실제 getTodayGratitudeDate() 결과를 먼저 확인할 것
```

### 트러블슈팅 메모

| 증상 | 원인 | 해결 |
|------|------|------|
| preview 서버가 구버전 빌드 서빙 | 경로 불일치 또는 캐시 | 포트 프로세스 종료 후 재시작 |
| streak이 예상과 다름 | 4AM 날짜 경계 | `getTodayGratitudeDate()` 반환값 확인 |
| Vercel `/api/` 접속 시 React 앱 표시 | Function 미배포 또는 routes 미적용 | Vercel 빌드 로그 + Functions 탭 확인 |
| TypeScript `process` 오류 (api/) | Edge Runtime에서 process 미지원 | Node.js Runtime 사용 (`export const config` 제거) |
| 배지 모달 미표시 | streak 증가 미발생 | `useAchievements`의 `streak > prevStreak` 조건 확인 |
