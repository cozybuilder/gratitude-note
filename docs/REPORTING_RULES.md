# REPORTING_RULES.md — 클로 완료 보고 형식 규칙

> 채팅방이 바뀌어도 완료 보고 형식이 일정하게 유지되도록 고정합니다.  
> 클로는 모든 완료 보고를 반드시 하나의 코드블록 안에 작성합니다.

---

## 완료 보고 형식

```text
━━━━━━━━━━━━━━━━━━
클로 → 코비 완료 보고
작업명: [작업명]
━━━━━━━━━━━━━━━━━━

① 수정 파일

② 추가 파일

③ 삭제 파일

④ 변경 이유

⑤ 검증 결과

⑥ 커밋 해시

⑦ push 성공 여부

⑧ 다음 작업 필요 여부
━━━━━━━━━━━━━━━━━━
```

---

## 적용 규칙

- 위 형식은 **코드블록(`text`) 하나**에 전부 담습니다.
- 항목이 해당 없으면 `없음`으로 명시합니다. 항목 자체를 생략하지 않습니다.
- 작업명은 코비 지시서의 `작업명:`을 그대로 사용합니다.
- 검증 결과에는 `npm run build`, `npx cap sync android`, curl 테스트 등 실제 수행 결과를 기록합니다.
- 커밋 해시는 전체 40자가 아닌 단축 해시(7자)로 기재합니다.
- push 성공 여부는 `✅ 성공` 또는 `❌ 실패 (사유: )`로 기재합니다.
- 문서 반영 검토는 보고 코드블록 **아래**에 별도 섹션으로 추가합니다.

---

## 문서 반영 검토 형식 (코드블록 아래에 추가)

```text
### 문서 반영 여부
- CHANGELOG.md: Yes / No (사유: )
- DECISIONS.md: Yes / No (사유: )
- PROJECT_OVERVIEW.md: Yes / No (사유: )
- RELEASE_CHECKLIST.md: Yes / No (사유: )
- MEMORY.md: Yes / No (사유: )
```
